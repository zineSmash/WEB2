반응형 디자인과 미디어 쿼리 소개 (링크)

<style> 태그에 아래와 같이 코드를 작성하고,

<style> div{
border:5px solid green;
}
@media(max-width:800px){
div{
display: inline;
}
}
@media(min-width:800px){
div{
display:block;
}
}
</style>

본문에는 아래와 같이 적어준다.

<div>>
Reponsive
</div>

Reponsive

브라우저의 크기를 조절해보자.
Responsive 텍스트 박스의 크기가 브라우저의 크기에 따라 바뀔 것이다.
이렇게 브라우저의 크기에 따라 요소가 달라지는 것을 반응형 디자인이라고 하고,
그것을 가능하게 해주는 것이 미디어 쿼리라는 것이다. (스타일 태그에서 작성했던 @media() 이것이 미디어 쿼리)

CSS 코드의 재사용 (링크)

지금까지 <style> 태그를 통해 본문을 꾸미는 작업을 해보았다.
각 페이지마다 각각 스타일 태그와 속성을 작성해서 본문을 꾸몄는데,
이러한 작업이 html을 공부하는 수준에서는 무리가 없는 작업이 되겠지만,
실제 서비스하는 웹페이지에서는 이런 작업 방식이 효율적일리가 없다.
같은 속성의 내용이 각 페이지마다 따로 작성되어 있기에 디자인적인 통일을 위해 어떤 부분을 수정한다고 하면,
각 페이지마다 일일이 수정해야하는 번거러움이 있을 것이다.
css 는 그럴 때 제힘을 발휘한다. (...라고 나는 생각한다)

앞서 작성했던 <style> 태그 안의 속성들을 조금 더 효율적으로 사용하도록 바꿔보자.

  • 먼저 style.css 라는 새로운 파일을 하나 만들자
  • 그리고 <style> 태그 안의 내용을 복사해서 style.css 파일에 붙여넣는다
  • style.css 에는 html 관련 기본 태그들은 필요없다, 복사한 내용만 넣으면 된다
  • 복사를 했다면 저장

다음은 style.css 를 웹페이지 본문에 적용을 해보자

  • 각 웹페이지의 <head> 태그 내부에 아래와 같이 써준다
  • <link rel="stylesheet" href="style.css">
  • link rel="stylesheet" 는 다음에 올 내용의 파일을 연동해 사용하겠다는 의미
  • href="style.css" 는 앞서 작성한 style.css 파일의 경로
  • 현재는 이정도로 이해하고 있으며, 나중에 좀 더 이해하게 되면 내용을 수정할지도 모르겠다

이렇게 작업을 해주고 웹페이지를 확인해보면, 모든 웹페이지에 동일한 스타일 속성들이 적용된 것을 볼 수 있다.
<link rel="stylesheet" href="style.css"> 으로 전체 웹페이지에 적용한 것과 별개로,
각 페이지마다 스타일 태그와 속성을 따로 추가해두면 그 페이지만의 속성을 적용할 수도 있다.
여러가지로 응용해서 익혀보도록 하자.
